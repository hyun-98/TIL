1. VPC 생성

[IPv4 CIDR 블록]
: IP 주소 범위를 표현하는 표기법
ex) 10.0.0.0/16
10.0.0.0 👉 시작 주소
/16 👉 네트워크 크기

| CIDR | IP 개수    | 용도 느낌         |
| ---- | -------- | ------------- |
| /16  | 약 65,000 | 큰 네트워크 (VPC)  |
| /24  | 256개     | 작은 네트워크 (서브넷) |
| /28  | 16개      | 아주 작은 단위      |

[IPv6 CIDR 블록]
: IPv6 주소 범위를 표현하는 방식
ex) 2001:db8::/32
2001:db8:: 👉 시작 주소
/32 👉 앞 32비트는 네트워크

👉 나머지 96비트는 자유롭게 사용
| 구분    | IPv4        | IPv6          |
| ----- | ----------- | ------------- |
| 주소 길이 | 32비트        | 128비트         |
| 주소 개수 | 제한적         | 거의 무한         |
| 예시    | 10.0.0.0/16 | 2001:db8::/32 |

[테넌시]
서버(물리 머신)를 다른 사람과 같이 쓰느냐, 혼자 쓰느냐를 결정하는 옵션

| 구분        | 기본값 (Default) | 전용 (Dedicated) |
| --------- | ------------- | -------------- |
| 물리 서버     | 공유            | 단독 사용          |
| 다른 고객과 공유 | O             | X              |
| 보안 격리     | 기본 수준         | 더 강함           |
| 비용        | 저렴            | 비쌈             |
| 사용 용도     | 일반 서비스        | 금융, 공공, 규제 환경  |
