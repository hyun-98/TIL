# 1. VPC 생성

## 📌 IPv4 CIDR 블록

IP 주소 범위를 표현하는 표기법

- 예시: `10.0.0.0/16`
  - `10.0.0.0` → 시작 주소
  - `/16` → 네트워크 크기

### CIDR별 크기

| CIDR | IP 개수     | 용도 |
|------|------------|------|
| /16  | 약 65,536개 | VPC (큰 네트워크) |
| /24  | 256개       | 서브넷 |
| /28  | 16개        | 최소 단위 |

---

## 📌 IPv6 CIDR 블록

IPv6 주소 범위를 표현하는 방식

- 예시: `2001:db8::/32`
  - `2001:db8::` → 시작 주소
  - `/32` → 앞 32비트는 네트워크

👉 나머지 96비트는 자유롭게 사용 가능

### IPv4 vs IPv6 비교

| 구분       | IPv4         | IPv6            |
|------------|-------------|-----------------|
| 주소 길이  | 32비트       | 128비트         |
| 주소 개수  | 제한적       | 거의 무한       |
| 예시       | 10.0.0.0/16  | 2001:db8::/32   |

---

## 📌 테넌시 (Tenancy)

물리 서버를 다른 사용자와 공유할지 여부를 결정하는 옵션

| 구분        | Default (기본) | Dedicated (전용) |
|------------|----------------|------------------|
| 물리 서버   | 공유           | 단독 사용        |
| 고객 간 공유 | O              | X                |
| 보안 수준   | 기본           | 높음             |
| 비용        | 저렴           | 비쌈             |
| 사용 사례   | 일반 서비스     | 금융, 공공, 규제 환경 |

---

# 📌 핵심 개념 정리

## 1. VPC
- AWS 내에서 사용하는 가상 네트워크
- CIDR로 IP 범위 정의

---

## 2. 서브넷
- VPC를 나눈 네트워크 단위
- 하나의 AZ에 속함

### 종류
- Public Subnet: 인터넷 직접 연결
- Private Subnet: 내부 전용

---

## 3. Internet Gateway (IGW)
- VPC ↔ 인터넷 연결

---

## 4. NAT Gateway
- Private Subnet → 인터넷 outbound 전용

---
