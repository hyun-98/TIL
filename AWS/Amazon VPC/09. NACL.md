nacl : 서브넷 단위에서 트래픽을 제어하는 방화벽 역할
-> 이 서브넷에 들어오고 나가는 트래픽을 허용할지/차단할지 결정하는 규칙 집합

하나의 서브넷에는 하나의 NACL만 연결 가능 (한개의 NACL로 여러개의 서브넷에 연결 가능) Subnet : NACL = N : 1

[NACL 실습 (SSH는 접속 가능, HTTP는 접속 불가]

1. 네트워크 ACL 생성

2. 인바운드 규칙 설정

SSH는 허용, HTTP는 불가

규칙번호 : 규칙번호가 가장 낮은 규칙부터 평가된다

100, SSH ,0.0.0.0/0 ,허용
110, HTTP, 0.0.0.0/0, 거부

3. 아웃바운드 규칙 설정

100, 모든 트래픽, 0.0.0.0/0, 허용

-> 이렇게 되면 SSH로는 접속이 가능하지만 HTTP로는 접속이 불가능하다
