# 📚 TIL - 7장 비밀 값 관리 (Secrets Management)

## 📌 핵심 요약

비밀번호, API Key, DB 접속 정보 같은 민감한 값은
👉 **절대 코드나 Git에 저장하면 안 된다**

---

# 🚨 1. 왜 비밀 값 관리가 중요한가?

## ❌ 잘못된 방법

* `.env` 파일을 Git에 업로드
* 코드에 직접 하드코딩

👉 문제점:

* Git 히스토리에 영구적으로 남음
* 유출 시 서비스 전체 위험
* 키 교체 비용 발생

---

## ⚠️ 유출 시 대응

👉 단순 삭제로 해결되지 않음

* 새로운 키 발급
* 기존 키 전면 폐기
* 관련 시스템 모두 수정 필요

---

# 🔐 2. 비밀 값 관리 방법

## ✔️ 대표적인 방법들

### 1️⃣ Blackbox

* Git에 암호화된 상태로 저장
* 복호화 키는 별도 관리

---

### 2️⃣ Vault

* HashiCorp Vault
* 중앙 집중형 비밀 관리 시스템
* 동적 비밀 생성 가능 (DB 계정 자동 생성 등)

---

### 3️⃣ AWS Secrets Manager

* AWS에서 제공하는 관리형 비밀 저장소
* 가장 많이 사용하는 방식 중 하나

---

# ☁️ 3. AWS Secrets Manager

## ✔️ 개념

* 비밀 값을 안전하게 저장하고
  👉 **필요할 때만 가져다 쓰는 방식**

---

## 🔑 저장 가능한 데이터

* DB 계정 정보
* API Key
* OAuth 토큰
* 인증서

---

# 🔐 4. Secrets Manager 동작 절차

```text id="secret1"
1. 관리자 → Secrets Manager에 비밀 등록
2. AWS KMS로 암호화
3. 서버에서 AWS CLI / SDK로 요청
4. IAM 권한 확인
5. 복호화된 값 반환
6. 애플리케이션에서 사용
```

---

## 🔄 단계별 상세 설명

### 1️⃣ 비밀 생성

* Key-Value 형태로 저장

```json
{
  "DB_USERNAME": "admin",
  "DB_PASSWORD": "1234"
}
```

---

### 2️⃣ 암호화 (KMS)

* AWS Key Management Service 사용
* 저장 시 자동 암호화

👉 개발자는 암호화 로직 신경 안 써도 됨

---

### 3️⃣ 서버 요청

* AWS CLI 또는 SDK 사용
* 특정 IAM Role로 접근

---

### 4️⃣ 권한 검증 (IAM)

* 요청한 주체가 접근 가능한지 확인
* 최소 권한 정책 적용

---

### 5️⃣ 복호화 및 반환

* Secrets Manager가 값을 복호화해서 전달

---

### 6️⃣ 애플리케이션 사용

* 메모리에만 유지 (파일 저장 X 권장)

---

# 🧠 핵심 구조 이해

```text id="secret-flow"
[Secrets Manager]
        ↑
   (KMS 암호화)
        ↓
[암호화된 비밀 저장]

        ↓ 요청
[EC2 / 서버]
        ↓ (IAM 인증)
[Secrets Manager 응답]
        ↓
[애플리케이션 사용]
```

---

# 💡 실무에서 중요한 포인트

## ✔️ 1. 환경 변수 vs Secrets Manager

| 구분 | 환경 변수 | Secrets Manager |
| -- | ----- | --------------- |
| 보안 | 낮음    | 높음              |
| 관리 | 어려움   | 중앙 관리           |
| 추천 | ❌     | ✅               |

---

## ✔️ 2. 캐싱 전략

* 매 요청마다 Secrets Manager 호출 ❌
* 애플리케이션 시작 시 1회 조회 후 캐싱 ✅

---

## ✔️ 3. 자동 로테이션 (중요 ⭐)

* 비밀 값 자동 변경 가능
* DB 비밀번호 자동 갱신

👉 보안 수준 크게 향상

---

## ✔️ 4. 접근 제어

* IAM Role 기반 접근
* IP 기반 제한 가능

---

# ⚠️ 주의사항

* 로그에 비밀 값 출력 금지
* 클라이언트(프론트)에 전달 금지
* `.env` 파일 Git 업로드 금지
