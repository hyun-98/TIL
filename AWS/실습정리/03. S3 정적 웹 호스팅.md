# 🌐 S3 정적 웹 호스팅이란?

Amazon S3는 원래 파일 저장 서비스인데,
이걸 웹 서버처럼 사용해서 **정적 사이트(HTML, CSS, JS)**를 그대로 공개할 수 있어요.

👉 즉:

* EC2 ❌ 필요 없음
* 서버 관리 ❌
* 파일 업로드만 하면 바로 웹사이트

---

# 🧱 구조 이해

```
사용자 → S3 URL → HTML 파일 반환
```

예시 URL:

```
http://your-bucket-name.s3-website-...amazonaws.com
```

---

# 🚀 실습 (처음부터 끝까지)

## 1️⃣ S3 버킷 생성

1. AWS 콘솔 → S3
2. 버킷 생성

   * 버킷 이름: **도메인처럼** (예: `my-site-123`)
   * 리전: 가까운 곳 선택 (서울 추천)

[버킷 네임스페이스]
> 글로벌 네임스페이스
👉 “인터넷 도메인 (example.com)”
> 리전 네임스페이스
👉 “서버 위치 (서울 데이터센터)”

[객체 소유권]
- Bucket owner enforced (ACL 비활성화됨)
: 모든 객체 = 무조건 버킷 소유자
- Bucket owner preferred (ACL 활성화됨)
: 기본은 업로더 소유

[퍼블릭 엑세스 차단 설정]
: Amazon S3에서 버킷이나 객체가 인터넷에 공개되는 걸 “강제로 막는 안전장치”

⚠️ 중요:

* "퍼블릭 액세스 차단" → **해제해야 함**

[버전 관리]
파일을 덮어써도 이전 버전을 계속 보관하는 기능

[태그]
리소스 관리용 메타데이터



---

## 2️⃣ 정적 웹 호스팅 활성화

버킷 → 속성(Properties) → 아래로 내려서:

👉 **Static website hosting 활성화**

* Index document:

```
index.html
```

* Error document (선택):

```
error.html
```

---

## 3️⃣ HTML 파일 업로드

예시 `index.html`:

```html
<!DOCTYPE html>
<html>
<head>
  <title>My S3 Website</title>
</head>
<body>
  <h1>Hello S3 Hosting 🚀</h1>
</body>
</html>
```

`style.css`
```
body {
  font-family: Arial, sans-serif;
  text-align: center;
  margin-top: 100px;
  background-color: #f4f4f4;
}

h1 {
  color: #333;
}

button {
  padding: 10px 20px;
  font-size: 16px;
  cursor: pointer;
}

#message {
  margin: 20px;
  font-size: 18px;
  color: blue;
}
```

`app.js`
```app.js
function changeText() {
  const msg = document.getElementById("message");
  msg.innerText = "🎉 버튼 클릭 성공! S3에서 잘 동작 중입니다!";
}
```
👉 S3에 업로드

---

## 4️⃣ 버킷 정책 설정 (퍼블릭 접근 허용)

버킷 → "권한(Permissions)" → "버킷 정책"

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicRead",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::YOUR_BUCKET_NAME/*"
    }
  ]
}
```

[버킷 정책]
“이 버킷에 누가 접근할 수 있는지 정의하는 규칙”

---

## 5️⃣ 접속 확인

S3 → Properties → Static Website Hosting URL 클릭

👉 이렇게 뜨면 성공:

```
http://your-bucket.s3-website-xxxx.amazonaws.com
```

# 🔥 (중요) 도메인 연결

S3 자체 URL 대신 도메인 붙이려면:

### 방법

* Amazon Route 53 사용
* 또는 외부 DNS

👉 핵심:

* CNAME 또는 Alias 레코드
* S3 웹사이트 endpoint 연결

---

# 🔒 HTTPS 적용 (중요 포인트)

❗ S3 단독으로는 HTTPS 지원 안 됨

👉 반드시 아래 필요:

* Amazon CloudFront

구성:

```
사용자 → CloudFront (HTTPS) → S3
```

---

## CloudFront 설정 간단 흐름

1. CloudFront 생성
2. Origin: S3 버킷
3. Viewer protocol: HTTPS
4. 인증서:

   * AWS Certificate Manager

👉 완료하면 HTTPS URL 생성됨

---

# 📦 전체 아키텍처

```
[사용자]
   ↓
CloudFront (HTTPS)
   ↓
S3 (정적 파일)
```

# 👍 장점

* 서버 관리 필요 없음
* 비용 매우 저렴
* 배포 빠름

# 👎 단점

* 로그인 처리 어려움 (백엔드 필요)
* SSR 불가

---

# 💡 실무 팁

* 캐싱 문제 해결:

  * CloudFront invalidation 사용
* SPA 라우팅:

  * error.html → index.html로 설정
