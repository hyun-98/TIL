# 🌐 EC2 도메인 연결 + HTTPS 설정 가이드

> 💡 목표: EC2 서버에 도메인을 연결하고 HTTPS까지 적용하기

---

## 🧭 전체 흐름

1. 도메인 구매  
2. DNS 설정 (도메인 → EC2 연결)  
3. 연결 확인  
4. HTTPS 적용 (Let's Encrypt)  
5. 자동 갱신 설정  

---

## 1️⃣ 도메인 구매

👉 아래 사이트 중 아무 곳에서 구매 가능

- 가비아
- 후이즈
- Namecheap
- GoDaddy

### 📌 예시
```

myserver123.com

```

---

## 2️⃣ DNS 설정 (핵심 ⭐)

👉 도메인을 EC2 퍼블릭 IP에 연결

### 📌 DNS 관리에서 설정

| 타입 | 값 |
|------|----|
| A 레코드 | EC2 퍼블릭 IP |

---

### 📌 설정 예시

```

Type: A
Name: @
Value: 43.203.xxx.xxx

```

👉 저장 후 약 1~10분 기다림 (전파 시간)

---

## 3️⃣ 연결 확인

👉 브라우저에서 접속

```

[http://내도메인](http://내도메인)

````

### ✅ 정상 동작
- EC2에서 설정한 웹 페이지가 뜨면 성공

---

## 4️⃣ HTTPS 적용 (Let's Encrypt)

👉 무료 SSL 인증서 사용

---

### 📌 EC2 접속 후 실행

#### 1) certbot 설치

```bash
sudo yum install -y certbot python3-certbot-apache
````

---

#### 2) 인증서 발급

```bash
sudo certbot --apache
```

---

### 📌 진행 과정

* 이메일 입력
* 도메인 입력
* HTTPS 적용 여부 선택

---

### 🎉 결과

* HTTPS 자동 적용
* Apache 설정 자동 변경

---

## 5️⃣ HTTPS 확인

👉 브라우저 접속

```
https://내도메인
```

### ✅ 성공 기준

* 주소창에 🔒 표시

---

## 🔄 자동 갱신 (중요)

👉 Let's Encrypt 인증서는 90일 유효

---

### 📌 갱신 테스트

```bash
sudo certbot renew --dry-run
```

👉 정상 실행되면 자동 갱신 OK

---

## ⚠️ 문제 해결 체크리스트

### ❌ 도메인 접속 안 됨

* DNS 설정 확인
* IP 주소 확인
* 전파 시간 대기

---

### ❌ HTTPS 적용 실패

* 포트 80 열려있는지 확인
* 도메인 연결 완료 여부 확인

---

### ❌ HTTPS 접속 안 됨

```bash
sudo systemctl restart httpd
```
