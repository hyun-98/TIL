# 📚 TIL - 6장 배포 자동화 (Deployment Automation)

## 📌 핵심 요약

배포 자동화는 사람이 직접 서버에 접속해서 배포하는 것이 아니라
👉 **도구를 통해 코드 변경 → 자동 빌드 → 자동 배포까지 수행하는 과정**

---

# 🤖 1. 배포 자동화란?

## ✔️ 개념

* 코드 변경이 발생하면
  👉 **자동으로 빌드 + 테스트 + 배포까지 진행**

---

## 🔄 기본 흐름

```text id="auto1"
1. 코드 push (GitHub)
2. 빌드 및 테스트 실행
3. 배포 서버로 전달
4. 자동 배포 진행
```

---

## 👍 장점

* 휴먼 에러 감소
* 배포 속도 향상
* 일관된 배포 환경 유지
* CI/CD 파이프라인 구축 가능

---

# 🔐 2. AWS IAM (Identity and Access Management)

## ✔️ 개념

* AWS 리소스 접근 권한을 관리하는 서비스
* **누가(Who) / 무엇을(What) / 어디까지(How much)** 할 수 있는지 제어

---

## 🔑 핵심 구성 요소

### 1️⃣ 사용자 (User)

* AWS에 접근하는 계정

### 2️⃣ 그룹 (Group)

* 여러 사용자 묶음

### 3️⃣ 역할 (Role)

* 특정 서비스에 권한 부여
  👉 (예: EC2가 S3에 접근)

### 4️⃣ 정책 (Policy)

* 권한 정의 JSON 문서

---

## 💡 실무 포인트

* ❌ 루트 계정 사용 금지
* ✅ 최소 권한 원칙 (Least Privilege)
* ✅ 배포 서버에는 IAM Role 사용

---

# 🚀 3. AWS CodeDeploy

## ✔️ 개념

* AWS에서 제공하는 **배포 자동화 서비스**
* EC2, Auto Scaling, Lambda 등에 배포 가능

---

## 🔄 배포 흐름

```text id="cd1"
1. 애플리케이션 업로드 (S3 or GitHub)
2. CodeDeploy가 배포 대상 서버에 전달
3. AppSpec.yml 기준으로 배포 수행
4. 배포 완료 및 상태 확인
```

---

## 🎯 주요 기능

* 롤링 배포 / 블루-그린 배포 지원
* 자동 롤백 기능
* 배포 상태 모니터링
* 배포 로그 확인

---

# 📄 4. AppSpec.yml

## ✔️ 개념

* 배포 절차를 정의하는 설정 파일
* 프로젝트 최상단에 위치

👉 **"배포 스크립트의 설계도"**

---

## 🧾 기본 구조

```yaml
version: 0.0
os: linux

files:
  - source: /
    destination: /home/ec2-user/app

hooks:
  BeforeInstall:
    - location: scripts/before_install.sh

  AfterInstall:
    - location: scripts/after_install.sh

  ApplicationStart:
    - location: scripts/start.sh
```

---

## ⚙️ 주요 섹션 설명

### ✔️ files

* 어떤 파일을 어디로 복사할지 정의

---

### ✔️ hooks

* 배포 단계별 실행할 스크립트 정의

---

## 🔄 대표 Hook 흐름

```text id="hook1"
BeforeInstall → AfterInstall → ApplicationStart → ValidateService
```

---

## 💡 실무 예시

* BeforeInstall → 기존 파일 삭제
* AfterInstall → 의존성 설치
* ApplicationStart → 서버 실행
* ValidateService → 정상 동작 확인

---

# 🛠️ 5. 기타 배포 자동화 도구

## ✔️ Jenkins

* 가장 대표적인 CI/CD 도구
* 플러그인 기반 확장성

👉 기업에서 많이 사용

---

## ✔️ Capistrano

* Ruby 기반 배포 자동화 도구
* SSH 기반으로 서버 배포 수행

---

## 💡 비교

| 도구         | 특징              |
| ---------- | --------------- |
| CodeDeploy | AWS 통합, 관리형 서비스 |
| Jenkins    | 커스터마이징 강력       |
| Capistrano | 간단한 서버 배포       |

---

# 🧠 전체 구조 한눈에 보기

```text id="flow-final"
[개발자]
   ↓ (git push)
[GitHub]
   ↓
[CI/CD 도구 (Jenkins / CodeDeploy)]
   ↓
[EC2 / 서버]
   ↓
[AppSpec.yml 실행]
   ↓
[서비스 배포 완료 🚀]
```

* 서버가 여러 대일수록 관리 어려움
