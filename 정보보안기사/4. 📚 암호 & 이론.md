
# 🔐📚 암호 & 이론 1페이지 요약 (정보보안기사)

---

## 🔐 1. 핵심 개념

👉 암호 이론 = **정보를 읽을 수 없게 바꾸고(암호화), 다시 복원(복호화)하는 기술**

목표:

* 기밀성 (Confidentiality)
* 무결성 (Integrity)
* 인증 (Authentication)
* 부인방지 (Non-repudiation)

👉 CIA + α(부인방지)

---

# 🧱 2. 암호 방식

## 📌 (1) 대칭키 암호

👉 같은 키로 암호화/복호화

* 빠름
* 대용량 데이터 적합
* 키 공유 문제 있음

### ✔️ 대표

* AES (현대 표준)
* DES (구식)
* 3DES

---

## 📌 (2) 비대칭키 암호

👉 공개키 / 개인키 2개 사용

* 공개키로 암호화 → 개인키로 복호화
* 느리지만 안전

### ✔️ 대표

* RSA
* ECC (타원곡선 암호)

---

## 📌 (3) 하이브리드 방식

👉 대칭 + 비대칭 혼합

* 실제 HTTPS 구조
* 빠르고 안전

---

# 🔑 3. 해시 함수

👉 단방향 암호 (복호화 불가)

### ✔️ 특징

* 입력 → 고정 길이 출력
* 조금만 바뀌어도 결과 완전 변경

### ✔️ 대표

* SHA-256
* SHA-3

👉 비밀번호 저장에 사용

---

# 🧂 4. 솔트(Salt)

👉 해시 강화 기술

* 비밀번호에 랜덤 값 추가
* 동일 비밀번호도 다른 해시값

👉 목적: Rainbow Table 공격 방지

---

# 🔐 5. 디지털 서명

👉 “누가 보냈는지 + 변조 여부 확인”

### 과정

* 해시 생성
* 개인키로 서명
* 공개키로 검증

👉 기능:

* 인증
* 무결성
* 부인방지

---

# 📜 6. PKI (공개키 기반 구조)

👉 인증서 기반 신뢰 시스템

구성:

* 인증기관 (CA)
* 인증서
* 공개키/개인키

👉 웹 HTTPS 기반

---

# 🌐 7. HTTPS / SSL/TLS

👉 웹 보안 프로토콜

* 데이터 암호화
* 서버 인증
* 무결성 보호

---

# 🧨 8. 공격 & 취약점

* 중간자 공격 (MITM)
* 패스워드 크래킹
* Rainbow Table
* Brute Force (무차별 대입)

---

# 🧩 9. 핵심 비교 정리

| 구분     | 특징              |
| ------ | --------------- |
| 대칭키    | 빠름 / 키 공유 문제    |
| 비대칭키   | 느림 / 안전         |
| 해시     | 단방향 / 복호화 불가    |
| 디지털 서명 | 인증 + 무결성 + 부인방지 |

---

# 🔥 10. 시험 핵심 키워드 TOP 5

👉 이것만 외우면 핵심

* AES / RSA
* SHA-256
* 디지털 서명
* 공개키 / 개인키
* Salt
