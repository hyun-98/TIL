# 🖥️ 시스템 보안

## 🔐 1. 핵심 개념

👉 시스템 보안 = **OS/서버 보호 + 접근 통제 + 로그 관리**

목표:

* 무단 접근 차단
* 권한 최소화
* 침해 탐지
* 흔적 관리

---

# 👤 2. 접근통제 (AAA)

## 📌 AAA 구조

* Authentication → 인증 (너 누구?)
* Authorization → 인가 (뭐 할 수 있음?)
* Accounting → 감사 (뭐 했음?)

---

## 📌 접근통제 모델

| 모델   | 특징            |
| ---- | ------------- |
| DAC  | 소유자가 권한 결정    |
| MAC  | 보안 등급으로 강제 통제 |
| RBAC | 역할(직무) 기반     |

👉 시험 핵심: **RBAC 많이 출제**

---

# 🔑 3. 계정/권한 관리

* 최소 권한 원칙 (Least Privilege)
* 계정 공유 금지
* 관리자 권한 최소화
* 주기적 권한 점검

---

# 🔐 4. 인증 보안

## ✔️ 인증 요소

* 지식: 비밀번호
* 소유: OTP, 토큰
* 생체: 지문, 얼굴

👉 핵심: **MFA (다중 인증)**

---

# 🧱 5. 운영체제 보안

* 커널 보호
* 프로세스 보호
* 메모리 보호
* 파일 시스템 보호

## ⚠️ 공격

* 버퍼 오버플로우
* 권한 상승 공격

---

# 🧨 6. 악성코드 대응

* 바이러스
* 웜
* 트로이목마
* 랜섬웨어

👉 대응:

* 백신
* 패치
* 행위 기반 탐지

---

# 📊 7. 로그 관리 (매우 중요)

## ✔️ 로그 종류

* 시스템 로그
* 접속 로그
* 보안 로그

## ✔️ 핵심

* 누가 / 언제 / 무엇을 했는지
* 위변조 방지
* 정기 점검

---

# 🧩 8. 시스템 보안 강화 (Hardening)

👉 시스템 보안 핵심 단어

* 불필요 서비스 제거
* 포트 최소화
* 패치 업데이트
* 보안 설정 강화

---

# 💾 9. 백업/복구

* 전체 백업
* 증분 백업
* 재해 복구 (DR)

👉 랜섬웨어 대비 필수

---

# 🔥 10. 시험 핵심 키워드 TOP 5

👉 이것만 외워도 점수 나옴

* 접근통제 (DAC / MAC / RBAC)
* 최소권한 원칙
* MFA 인증
* 로그 관리
* 시스템 하드닝

---

# 🧠 초압축 한 줄 요약

👉 **시스템 보안 = “접근은 막고, 권한은 줄이고, 기록은 남기는 것”**
