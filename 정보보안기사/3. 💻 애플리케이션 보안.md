
# 💻 애플리케이션 보안 1페이지 요약 (정보보안기사)

---

## 🔐 1. 핵심 개념

👉 애플리케이션 보안 = **프로그램(웹/앱) 자체의 취약점을 막는 보안**

목표:

* 입력값 검증
* 코드 취약점 제거
* 데이터 보호
* 인증/세션 안전화

---

# 🧱 2. 주요 보안 영역

## 📌 (1) 입력값 검증

👉 사용자 입력을 믿지 않는다

* SQL Injection 방지
* XSS 방지
* Command Injection 방지

👉 핵심: **모든 입력은 공격 가능**

---

## 📌 (2) 인증/인가 보안

* 인증(Authentication) → 로그인 확인
* 인가(Authorization) → 권한 확인

### 문제점

* 세션 탈취
* IDOR (권한 우회)

---

## 📌 (3) 세션 관리

* 세션 하이재킹 방지
* 세션 타임아웃 설정
* 쿠키 보안 설정 (HttpOnly, Secure)

---

# 🧨 3. 주요 웹 공격

## 📌 대표 공격 TOP

* SQL Injection → DB 조작
* XSS → 스크립트 삽입
* CSRF → 사용자 권한으로 요청 위조
* File Upload 공격 → 악성파일 업로드
* Path Traversal → 경로 탈출 공격

---

# 🔐 4. 방어 기술

## 📌 입력 방어

* 입력값 검증 (Validation)
* 출력 인코딩 (Encoding)
* Prepared Statement 사용

---

## 📌 인증 방어

* MFA (다중 인증)
* 비밀번호 암호화 (해시 + 솔트)
* 로그인 시도 제한

---

## 📌 세션 방어

* 세션 ID 랜덤 생성
* HTTPS 사용
* 세션 만료 시간 설정

---

# 🧱 5. 개발 보안 (Secure Coding)

👉 안전한 코드 작성 원칙

* 입력값 검증 필수
* 에러 메시지 최소화
* 하드코딩 금지 (비밀번호 등)
* 최소 권한 적용

---

# 📊 6. 보안 테스트

* 취약점 점검
* 침투 테스트 (Penetration Test)
* 코드 리뷰
* 정적 분석 / 동적 분석

---

# 🧩 7. 보안 프레임워크

* OWASP Top 10 (웹 취약점 기준)
* SDL (Secure Development Lifecycle)

---

# 🔥 8. 시험 핵심 키워드 TOP 5

👉 이것만 외우면 핵심

* SQL Injection
* XSS
* CSRF
* 세션 하이재킹
* 입력값 검증

---
