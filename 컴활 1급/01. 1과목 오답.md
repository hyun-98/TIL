## 공개키 암호화 기법

두 개의 서로 다른 키(공개 키 + 개인 키) 사용 / 비대칭키

1. 기밀성

공개키로 암호화, 개인키로 복호화

2. 인증, 전자서명

개인키로 서명, 공개키로 검증

3. 대표 알고리즘

RSA algorithm, ECC (타원곡선 암호), Diffie-Hellman

특징 
* 알고리즘이 복잡하며 암호화와 복호화 속도가 느리다
* 키의 분배가 용이하고, 관리해야 할 키의 수가 적다
(DES : 대칭키 암호화 알고리즘)

---
## BIOS
: 컴퓨터 전원을 켰을 때 가장 먼저 실행되는 펌웨어

역할
* 하드웨어 점검 (POST)
* 부팅장치 찾기 (OS, SSD,HDD,usb 찾음)
* OS 실행

특징
* 저장이 꺼져도 유지되며, 메인보드 ROM에 저장

---
## 네트워크 장비 브리지(Bridge)
: 2개 이상의 네트워크를 연결하면서 데이터 흐름을 관리하는 장비

역할
* 네트워크 연결 (서로 다른 LAN을 하나처럼 연결)
* 트래픽 필터링 (모든 트래픽 무조건 보낸다 X, MAC 주소 기반 필요한 곳에만 보낸다)
* 충돌 감소
* OSI 참조 모델의 데이터링크 계층에 해당

---
## SSD (회전 없이 반도체로 데이터를 저장)
: 속도가 매우 빠르고 충격에 강하다 (소음 없고, 발열에 강하다)

---
## ASCII 코드
: 컴퓨터에서 문자를 숫자로 표현하기 위한 표준 코드
* 7비트 사용
* 128개 문자 표현 가능 (영문자, 숫자, 특수문자 포함)

---
## 컴퓨터 운영체제의 성능 평가 기준
1. 처리 능력(Throughput) : 일정 시간 동안 처리하는 작업의 양
2. 반환 시간(Turnaround Time) : 작업을 시작해서 끝날 때까지 걸리는 총 시간
3. 응답 시간( Response Time) : 사용자가 요청한 후 첫 반응이 나올 때까지 시간
4. 사용 가능도(Availability) : 시스템이 정상적으로 사용 가능한 시간 비율
5. 신뢰도(Reliability) : 시스템이 오류 없이 안정적으로 동작하는 정도
