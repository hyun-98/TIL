## 공개키 암호화 기법

두 개의 서로 다른 키(공개 키 + 개인 키) 사용 / 비대칭키

1. 기밀성

공개키로 암호화, 개인키로 복호화

2. 인증, 전자서명

개인키로 서명, 공개키로 검증

3. 대표 알고리즘

RSA algorithm, ECC (타원곡선 암호), Diffie-Hellman

특징 
* 알고리즘이 복잡하며 암호화와 복호화 속도가 느리다
* 키의 분배가 용이하고, 관리해야 할 키의 수가 적다
(DES : 대칭키 암호화 알고리즘)

---
## BIOS
: 컴퓨터 전원을 켰을 때 가장 먼저 실행되는 펌웨어 (ROM에 저장)

역할
* 하드웨어 점검 (POST)
* 부팅장치 찾기 (OS, SSD,HDD,usb 찾음)
* OS 실행

특징
* 저장이 꺼져도 유지되며, 메인보드 ROM에 저장

업데이트 하는 이유
* 하드웨어 호환성 개선
* 시스템 안정성 향상 / 보안 취약점 수정

---
## 네트워크 장비 브리지(Bridge)
: 2개 이상의 네트워크를 연결하면서 데이터 흐름을 관리하는 장비

역할
* 네트워크 연결 (서로 다른 LAN을 하나처럼 연결)
* 트래픽 필터링 (모든 트래픽 무조건 보낸다 X, MAC 주소 기반 필요한 곳에만 보낸다)
* 충돌 감소
* OSI 참조 모델의 데이터링크 계층에 해당

---
## SSD (회전 없이 반도체로 데이터를 저장)
: 속도가 매우 빠르고 충격에 강하다 (소음 없고, 발열에 강하다)

## ASCII 코드
: 컴퓨터에서 문자를 숫자로 표현하기 위한 표준 코드
* 7비트 사용
* 128개 문자 표현 가능 (영문자, 숫자, 특수문자 포함)

---
## 컴퓨터 운영체제의 성능 평가 기준
1. 처리 능력(Throughput) : 일정 시간 동안 처리하는 작업의 양
2. 반환 시간(Turnaround Time) : 작업을 시작해서 끝날 때까지 걸리는 총 시간
3. 응답 시간( Response Time) : 사용자가 요청한 후 첫 반응이 나올 때까지 시간
4. 사용 가능도(Availability) : 시스템이 정상적으로 사용 가능한 시간 비율
5. 신뢰도(Reliability) : 시스템이 오류 없이 안정적으로 동작하는 정도

## 렌더링
: 3D 모델이나 데이터 -> 실제 그림
* 빛, 그림자, 질감(텍스처), 색상, 시점 계산

주요 렌더링 기법
* 와이어프레임 : 뼈대만 표현
* 표면 렌더링 : 표면까지 표현
* 쉐이딩 : 빛과 그림자 표현
* 레이트레이싱 : 빛의 경로를 계산해서 표현

실시간 렌더링 (빠름, 게임), 비실시간 렌더링 (느리지만 고품질, 영화, CG)

## 보안 위협
* 바이러스 : 파일에 기생해서 퍼지는 악성 프로그램 // 데이터 손상
* 키로거 : 사용자가 키보드로 입력하는 내용을 몰래 기록하는 프로그램 또는 장치 // 정보 훔침
* 트로이목마 : 정상 프로그램처럼 보이지만, 실제로는 악성 기능을 숨기고 있는 프로그램 // 정보 탈취
* 스파이웨어 : 사용자의 동의 없이 개인정보나 사용 기록을 몰래 수집하는 악성 프로그램 // 유출

## 아날로그(속도계, 온도계) vs 디지털 (컴퓨터, 스마트폰)
| 구분    | 아날로그  | 디지털      |
| ----- | ----- | -------- |
| 데이터   | 연속값   | 이산값(0,1) |
| 정확도   | 낮음    | 높음       |
| 처리 방식 | 물리량   | 논리 연산    |
| 활용    | 특정 목적 | 범용       |

## 그래픽 파일의 형식 (비트맵)
* JPEG : 사진용 이미지에 최적 (손실 압축, 색상 풍부)
* BMP : 무압축, 비효율 압축, 화질 최고, 용량 최악
* GIF : 무손실 압축, 256 색, 움직이는 이미지 가능
* PNG : 무손실 압축, 투명 색상 지원, 고화질

<-> 벡터 방식
* 베지어, 스플라인 등의 곡선을 이용하여 이미지를 표현하므로 확대/축소 시 화질의 손상이 거의 없다

---
## 통신
* LWPA : 저전력 + 장거리 통신
* LTE : 빠른 데이터 속도, '스마트폰 인터넷'
* Wi-fi : 근거리 무선 인터넷 (무선 LAN 기술)
* USN : 센서 네트워크 (스마트 시티, 환경 감지)

---
## 캐시 메모리
: CPU와 주기억장치(RAM) 사이에 있는 고속 임시 저장 공간
캐시 메모리는 CPU의 처리 속도를 높이기 위한 고속 임시 저장 공간이다 (매우 빠르지만, 용량 비싸다)

---
## 프록시(froxy)
: 사용자 대신 인터넷에 요청을 보내주는 중간 서버
* 사용자 대신 요청
* 캐싱(자주 요청한 데이터 저장) + 보안 기능
* 네트워크 병목현상 해결

---
## JAVA
* 네트워크 환경에서 분산 작업 가능
* 멀티스레드 기능을 제공, 여러 작업이 동시에 처리 가능
* 운영체제와 관계없이 독립적으로 실행 할 수 있는 프로그램 작성

<-> LISP : 수식처리 + 기호처리 + AI

## SRAM vs DRAM

| 구분  | SRAM  | DRAM   |
| --- | ----- | ------ |
| 속도  | 빠름    | 느림     |
| 재충전 | 필요 없음 | 필요 있음  |
| 가격  | 비쌈    | 저렴     |
| 용도  | 캐시    | 메인 메모리 |

* 블루투스: 짧은 거리에서 기기끼리 무선 연결
