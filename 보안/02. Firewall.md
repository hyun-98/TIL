# 🛡️ F/W (Firewall)

* 네트워크 경계에서 트래픽을 필터링
* 허용/차단 정책 기반 접근 통제

### 📦 방화벽은 뭘 검사하냐?
- 어디서 왔는지 (IP 주소)
- 어떤 문으로 들어오는지 (포트 번호)
- 어떤 종류의 요청인지

### 방화벽의 종류
#### 1. 네트워크 방화벽 (AWS Security Group)
ex. EC2 SG
   
#### 2. 호스트 방화벽 (Window Defneder firewall)
-> 내 컴퓨터 안에서 막는다
   
#### 3. WAF (Web Application Firewall) (ex. AWS WAF)
-> 웹공격만 전문적으로 막는다

* HTTP : 암호화 없는 통신
* HTTPS : 암호화 된 통신
(SSL/TLS : HTTPS를 만드는 암호화 기술 자체)

* WAF는 HTTPS가 암호화 되어 있어 읽지 못하는데, SSL termination이라는 기술로 암호화 된 내용을 읽어본 후 검사한 후 봉인해서 다시 서버로 보낸다


* WAF가 구별하는 방법

방법 1: 📋 블랙리스트 방식

방법 2: ✅ 화이트리스트 방식

방법 3: 🤖 AI/머신러닝 방식 (요즘 WAF!)


### 🗡️ WAF가 처리한는 공격들
#### 1. SQL Injection
-> 데이터베이스 해킹 시도

```sql
-- 위험한 방식 (입력값을 그냥 붙여버림)
"SELECT * FROM users WHERE id='" + 입력값 + "'"

-- 안전한 방식 (입력값은 그냥 문자로만 처리)
SELECT * FROM users WHERE id=?
```
* 입력값에 SQL 명령어를 입력하여 그것도 인식되어 장애 발생


#### 2. XSS (스크립트 공격)
-> 웹페이지에 악성 코드 삽입 / 읽기만 해도 피해 발생
```html
<!-- 위험: 그대로 화면에 출력 -->
<div> 사용자입력값 </div>

<!-- 안전: 특수문자를 무력화시킴 -->
<script> → &lt;script&gt; 로 변환해서 출력
```

#### 3. 🗡️ DDoS (Distributed Denial of Service)
* 다수의 공격자가 동시에 트래픽을 보내 서비스 마비
* 서버 과부하 유발 → 정상 사용자 접속 불가

🤖 실제 공격은 어떻게 하냐?

- 해커는 바이러스 감염된 컴퓨터들을 모음 (좀비 PC)
- 한 번에 서버로 요청 보내기

👉 이런 좀비 PC 네트워크를 **Botnet**이라고 해

🛡️ 막는법
1. 문 앞에 경비 세우기
   -> 요청들을 걸러낸다
2. 사람인지 로봇인지 확인 CAPTCHA (로그인 인증)
3. 서버 여러 개 사용
4. 트래픽 필터 서비스 (AWS shield, Cloudflare)
5. 속도 제한 -> redis로 많이 구현하며, 예시로는 한 IP 1분에 1000번만 허용


4. 파일 인젝션 (File Injection)
