## 정보보안 위험 관리의 4가지 대응 전략 (EOS 상황)
1. 순응
위험을 그대로 받아들임
- 비용 대비 위험이 작을 때
- 대응 비용이 더 클 때

-> 그냥 사용

2. 회피
👉 위험이 있는 행동 자체를 안 함
- 위험 자체를 제거

ex) 보안 취약한 시스템 폐기 / 위험한 서비스 종료

-> 바로 폐기

3. 전가
👉 위험을 다른 곳으로 넘김 (보험, 외주, 계약)
'문제 생기면 다른쪽 책임'

-> 서버를 이전 or 보안 관리를 외주에게 맡김

4. 감소
👉 위험을 줄이는 조치

->  방화벽 강화, 네트워크 분리 등 위험을 낮춘다.

# 🌐 네트워크 보안

## 🗡️ DDoS (Distributed Denial of Service)
* 다수의 공격자가 동시에 트래픽을 보내 서비스 마비
* 서버 과부하 유발 → 정상 사용자 접속 불가

🤖 실제 공격은 어떻게 하냐?

- 해커는 바이러스 감염된 컴퓨터들을 모음 (좀비 PC)
- 한 번에 서버로 요청 보내기

👉 이런 좀비 PC 네트워크를 **Botnet**이라고 해

🛡️ 막는법
1. 문 앞에 경비 세우기
   -> 요청들을 걸러낸다
2. 사람인지 로봇인지 확인 CAPTCHA (로그인 인증)
3. 서버 여러 개 사용
4. 트래픽 필터 서비스 (AWS shield, Cloudflare)
5. 속도 제한 -> redis로 많이 구현하며, 예시로는 한 IP 1분에 1000번만 허용

---

## 🛡️ F/W (Firewall)

* 네트워크 경계에서 트래픽을 필터링
* 허용/차단 정책 기반 접근 통제

📦 방화벽은 뭘 검사하냐?
- 어디서 왔는지 (IP 주소)
- 어떤 문으로 들어오는지 (포트 번호)
- 어떤 종류의 요청인지

방화벽의 종류
1. 네트워크 방화벽 (AWS Security Group)

   
2. 호스트 방화벽 (Window Defneder firewall)
-> 내 컴퓨터 안에서 막는다

EC2 보안 그룹으로 기본적인 접근 제어를 하고,
호스트 방화벽으로 Redis나 DB 포트를 내부 IP만 허용하도록 설정해
보안을 강화할 수 있습니다
   
3. WAF (Web Application Firewall) (ex. AWS WAF)
-> 웹공격만 전문적으로 막는다

* HTTP : 암호화 없는 통신
* HTTPS : 암호화 된 통신
(SSL/TLS : HTTPS를 만드는 암호화 기술 자체)

* WAF는 HTTPS를 읽지 못한다(암호화되어있으니까)
SSL termination이라는 기술로 암호화 된 내용을 읽어본 후 검사한 후
봉인해서 다시 서버로 보낸다


WAF가 구별하는 방법

방법 1: 📋 블랙리스트 방식

방법 2: ✅ 화이트리스트 방식

방법 3: 🤖 AI/머신러닝 방식 (요즘 WAF!)


🗡️ 공격들
1. SQL Injection
-> 데이터베이스 해킹 시도


2. XSS (스크립트 공격)
-> 웹페이지에 악성 코드 삽입

3. DDos 공격

4. 파일 인젝션 (File Injection)

# OSI 7 계층 (네트워크 통신 구조 모델)

OSI 7 계층은 네트워크 통신 과정을 7단계로 나눈 표준 모델입니다.

---

## 📌 L7. Application Layer (응용 계층)
- 프로토콜: HTTP, FTP, SMTP
- 사용자와 직접 상호작용하는 계층
- 웹 브라우저, 이메일 등 사용

👉 사용자가 네트워크를 실제로 사용하는 단계

---

## 📌 L6. Presentation Layer (표현 계층)
- 데이터 형식 변환
- 암호화 / 복호화
- 압축 / 인코딩 처리

👉 데이터를 읽을 수 있는 형태로 변환

---

## 📌 L5. Session Layer (세션 계층)
- 통신 세션 생성, 유지, 종료
- 대화(연결) 관리

👉 “대화 시작 ~ 끝” 관리

---

## 📌 L4. Transport Layer (전송 계층)
- 프로토콜: TCP, UDP
- 데이터 전달의 신뢰성 보장
- 오류 제어, 흐름 제어

👉 데이터를 안전하게 전달하는 역할

---

## 📌 L3. Network Layer (네트워크 계층)
- 프로토콜: IP, ICMP
- 라우팅 (경로 선택)
- 논리적 주소(IP) 기반 전달

👉 어디로 보낼지 결정

### 🔹 ICMP (Internet Control Message Protocol)
- IP 통신 중 문제 상황 보고 및 진단
- 네트워크 상태 확인용 프로토콜 (예: ping)

---

## 📌 L2. Data Link Layer (데이터 링크 계층)
- MAC 주소 기반 통신
- 같은 네트워크 내부에서 데이터 전달
- 오류 검출 기능 포함

👉 같은 네트워크 안에서 데이터 전달

---

## 📌 L1. Physical Layer (물리 계층)
- 실제 전기 신호, 광신호, 케이블
- 비트(0, 1) 전송

👉 물리적인 매체를 통해 데이터 전달


---



## 📌 L3 vs L7

### 🔹 L3 (네트워크 계층)

* IP 기반 처리
* 주소 기반 라우팅

### 🔹 L7 (응용 계층)

* Payload(내용) 분석
* HTTP, 사용자 데이터까지 검사

# 🛡️ 엔드포인트 & 내부 보안

## 📌 EDR (Endpoint Detection & Response)

* PC/서버의 위협 탐지 및 대응
* 행위 기반 분석 (악성코드 탐지)

---

## 📌 NAC (Network Access Control)

* 네트워크 접속 제어
* 인증된 사용자/기기만 접근 허용

---

## 📌 DLP (Data Loss Prevention)

* 데이터 유출 방지
* USB, 이메일, 클라우드 전송 통제

---

## 📌 DRM (Digital Rights Management)

* 문서/콘텐츠 접근 권한 제어
* 열람, 복사, 출력 제한

---

# 🚨 공격 및 위협

* 장기간 잠복 → 정보 탈취

# 📍 위치정보 법

## 📌 위치정보사업 vs 위치기반서비스사업

### 🔹 위치정보사업

* 위치정보를 **수집/제공**
* 허가 필요

### 🔹 위치기반서비스사업

* 위치정보를 활용한 서비스 제공
* 신고 대상

👉 핵심:

* 수집/제공 = 사업자
* 활용 = 서비스사업자

---

# 🔐 개인정보 개념

## 📌 가명정보 vs 익명정보

### 🔹 가명정보

* 일부 정보 제거/대체
* 추가 정보 있으면 재식별 가능

👉 예: 이름 → 코드화

---

### 🔹 익명정보

* 완전히 식별 불가능
* 복원 불가

👉 개인 식별 절대 불가
