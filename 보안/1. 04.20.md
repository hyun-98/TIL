## VAT Vulnerability Assessment Tool
👉 시스템/네트워크의 보안 취약점을 자동으로 찾아주는 도구


## 정보보안 위험 관리의 4가지 대응 전략 (EOS 상황)
1. 순응
위험을 그대로 받아들임
- 비용 대비 위험이 작을 때
- 대응 비용이 더 클 때

-> 그냥 사용

2. 회피
👉 위험이 있는 행동 자체를 안 함
- 위험 자체를 제거

ex) 보안 취약한 시스템 폐기 / 위험한 서비스 종료

-> 바로 폐기

3. 전가
👉 위험을 다른 곳으로 넘김 (보험, 외주, 계약)
'문제 생기면 다른쪽 책임'

-> 서버를 이전 or 보안 관리를 외주에게 맡김

4. 감소
👉 위험을 줄이는 조치

->  방화벽 강화, 네트워크 분리 등 위험을 낮춘다.

# 🌐 네트워크 보안

## 📌 DDoS (Distributed Denial of Service)

* 다수의 공격자가 동시에 트래픽을 보내 서비스 마비
* 서버 과부하 유발 → 정상 사용자 접속 불가

---

## 📌 F/W (Firewall)

* 네트워크 경계에서 트래픽을 필터링
* 허용/차단 정책 기반 접근 통제

---

## 📌 IPS (Intrusion Prevention System)

* 침입을 **탐지 + 차단**
* 주요 탐지 대상:

  * FTP
  * HTTP

---

## 📌 WAF (Web Application Firewall)

* 웹 애플리케이션 보호
* 주요 보호 대상:

  * HTTP / HTTPS
  * SSL/TLS 기반 통신
* SQL Injection, XSS 방어

---

## 📌 L3 vs L7

### 🔹 L3 (네트워크 계층)

* IP 기반 처리
* 주소 기반 라우팅

### 🔹 L7 (응용 계층)

* Payload(내용) 분석
* HTTP, 사용자 데이터까지 검사

---

# 🛡️ 엔드포인트 & 내부 보안

## 📌 EDR (Endpoint Detection & Response)

* PC/서버의 위협 탐지 및 대응
* 행위 기반 분석 (악성코드 탐지)

---

## 📌 NAC (Network Access Control)

* 네트워크 접속 제어
* 인증된 사용자/기기만 접근 허용

---

## 📌 DLP (Data Loss Prevention)

* 데이터 유출 방지
* USB, 이메일, 클라우드 전송 통제

---

## 📌 DRM (Digital Rights Management)

* 문서/콘텐츠 접근 권한 제어
* 열람, 복사, 출력 제한

---

# 🚨 공격 및 위협

## 📌 APT (Advanced Persistent Threat)

* 지능형 지속 공격
* 장기간 잠복 → 정보 탈취

---

# 📊 개인정보 & 인증

## 📌 PIA (Privacy Impact Assessment)

* 개인정보 영향평가
* 시스템 도입 시 개인정보 위험 분석

---

## 📌 ISMS-P

* 정보보호 + 개인정보보호 통합 인증
* 기업 보안 관리체계 인증 제도

---

# 🌍 네트워크 / 국제

## 📌 CSPF (Constraint-based SPF)

* 제약 조건 기반 최단 경로 탐색
* QoS, 대역폭 고려

---

## 📌 EIGRP

* Cisco 고유 라우팅 프로토콜
* 거리 벡터 + 링크 상태 혼합

---

## 📌 GDPR

* EU 개인정보 보호법
* 개인정보 처리 시 강력한 규제 적용

---

# 📍 위치정보 법

## 📌 위치정보사업 vs 위치기반서비스사업

### 🔹 위치정보사업

* 위치정보를 **수집/제공**
* 허가 필요

### 🔹 위치기반서비스사업

* 위치정보를 활용한 서비스 제공
* 신고 대상

👉 핵심:

* 수집/제공 = 사업자
* 활용 = 서비스사업자

---

# 🔐 개인정보 개념

## 📌 가명정보 vs 익명정보

### 🔹 가명정보

* 일부 정보 제거/대체
* 추가 정보 있으면 재식별 가능

👉 예: 이름 → 코드화

---

### 🔹 익명정보

* 완전히 식별 불가능
* 복원 불가

👉 개인 식별 절대 불가
